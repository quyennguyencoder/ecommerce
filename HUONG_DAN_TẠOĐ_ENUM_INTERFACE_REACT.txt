PHẦN 2: INTERFACE DTO CẦN CHUYỂN ĐỔI
================================================================================

2.1 DTO CHUNG: ApiResponse<T>
Vị trí Java: com.nguyenquyen.ecommerce.dto.ApiResponse

Cấu trúc Java:
{
  status: HttpStatus,      // HTTP Status (200, 400, 500, etc.)
  message: String,         // Thông báo phản hồi
  data: T                   // Generic data payload
}

Yêu cầu chuyển đổi:
- Tạo TypeScript generic interface ApiResponse<T>
- Status nên là string type hoặc enum với các giá trị HTTP (SUCCESS, ERROR, etc.)
- Message là chuỗi (string)
- Data là generic type (T)

---

2.2 DTO CHUNG: JwtInfo
Vị trí Java: com.nguyenquyen.ecommerce.dto.JwtInfo

Cấu trúc Java:
{
  id: String,          // JWT ID (UUID)
  userId: Long,        // ID của người dùng
  subject: String,     // Email hoặc số điện thoại
  roles: String,       // Tên role của người dùng
  issuedAt: Date,      // Thời gian cấp token
  expiresAt: Date      // Thời gian hết hạn token
}

Yêu cầu chuyển đổi:
- Tạo TypeScript interface JwtInfo
- id: string (UUID)
- userId: number
- subject: string (email hoặc phone)
- roles: string
- issuedAt: Date | string (ISO 8601)
- expiresAt: Date | string (ISO 8601)

---

2.3 DTO AUTH: LoginRequest
Vị trí Java: com.nguyenquyen.ecommerce.dto.request.AuthLoginRequest

Cấu trúc Java:
{
  username: String,    // Email hoặc số điện thoại (bắt buộc)
  password: String     // Mật khẩu (bắt buộc)
}

Yêu cầu chuyển đổi:
- Tạo TypeScript interface LoginRequest
- username: string (email hoặc phone)
- password: string
- Cả hai field đều bắt buộc

---

2.4 DTO AUTH: LoginResponse
Vị trí Java: com.nguyenquyen.ecommerce.dto.response.LoginResponse

Cấu trúc Java:
{
  accessToken: String,     // JWT access token
  refreshToken: String,    // JWT refresh token
  tokenType: String,       // Loại token (Bearer)
  expiresIn: Long,         // Thời gian hết hạn (milliseconds)
  user: UserResponse       // Thông tin người dùng đã đăng nhập
}

Yêu cầu chuyển đổi:
- Tạo TypeScript interface LoginResponse
- accessToken: string
- refreshToken: string
- tokenType: string (thường là "Bearer")
- expiresIn: number (milliseconds)
- user: UserResponse (nested interface)

---

2.5 DTO USER: UserResponse
Vị trí Java: com.nguyenquyen.ecommerce.dto.response.UserResponse

Cấu trúc Java:
{
  id: Long,
  email: String,
  phone: String,
  name: String,
  dob: LocalDate,              // Ngày sinh
  address: String,
  avatar: String,              // URL ảnh đại diện
  active: Boolean,
  createdAt: LocalDateTime,
  updatedAt: LocalDateTime,
  role: RoleResponse           // Nested DTO
}

Yêu cầu chuyển đổi:
- Tạo TypeScript interface UserResponse
- id: number
- email: string
- phone: string
- name: string
- dob: Date | string (ISO 8601 format: YYYY-MM-DD)
- address: string
- avatar: string (URL)
- active: boolean
- createdAt: Date | string (ISO 8601)
- updatedAt: Date | string (ISO 8601)
- role: RoleResponse (nested interface)

---

2.6 DTO ROLE: RoleResponse
Vị trí Java: com.nguyenquyen.ecommerce.dto.response.RoleResponse

Cấu trúc Java:
{
  id: Long,

}

Yêu cầu chuyển đổi:
- Tạo TypeScript interface RoleResponse
- id: number

  utils/

PHẦN 4: YÊU CẦU CHI TIẾT CHO GITHUB COPILOT
================================================================================

4.1 YÊU CẦU CHUNG:
- Sử dụng TypeScript với strict mode
- Tất cả fields đều bắt buộc trừ khi được chỉ định là optional (dùng ?)
- Sử dụng camelCase cho tên properties
- Thêm comments giải thích cho mỗi field

4.2 ENUM CONVERSION:
- Tạo TypeScript enums hoặc const objects
- Nên tạo utility function để get label (tiếng Việt) từ enum value
- Ví dụ: getGenderLabel('MALE') → 'Nam'
- Nếu có thông tin màu/icon, hãy thêm vào

4.3 INTERFACE CONVERSION:
- Chuyển Java types sang TypeScript types:
  * Long → number
  * String → string
  * Boolean → boolean
  * LocalDate/LocalDateTime → Date | string
  * Nested objects → nested interfaces

- Các optional fields (có thể null/undefined):
  * Đánh dấu bằng ? hoặc | null | undefined

4.4 ADDITIONAL UTILITIES:
- Tạo helper functions nếu cần:
  * Parse/format dates
  * Validate enum values
  * Type guards cho interfaces

4.5 COMMENTS VÀ DOCUMENTATION:
- Thêm JSDoc comments cho mỗi interface/enum
- Ghi chú về source (file Java gốc)
- Ghi chú về ý nghĩa của mỗi field
